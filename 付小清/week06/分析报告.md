# 文本分类训练方法对比分析报告

> 数据集：CLUE/TNEWS（15 类中文新闻标题）  
> 实验环境：`conda` 环境 **py312**（`D:\Program\anaconda3\envs\py312`），PyTorch 2.5.1+cu121，CUDA 可用  
> 报告生成日期：2026-05-29

---

## 一、实验目的

在同一套 TNEWS 数据上，对比 **5 种文本分类训练/推理范式** 的效果与适用场景：

| 编号 | 方法 | 代码入口 | 是否需训练 |
|------|------|----------|------------|
| 1 | BERT + CLS 池化 | `src/train.py --pool cls` | 是 |
| 2 | BERT + Mean 池化 | `src/train.py --pool mean` | 是 |
| 3 | BERT + Max 池化 | `src/train.py --pool max` | 是 |
| 4 | LLM Zero-shot | `src_llm/classify_llm.py` | 否 |
| 5 | LLM SFT（LoRA） | `src_llm/train_sft.py` + `evaluate_sft.py` | 是 |

另做 **类别加权 Loss** 消融（`--use_class_weight`），观察不均衡数据下的指标变化。

## 二、数据概况

| 指标 | 数值 |
|------|------|
| 训练集 | 53,360 条 |
| 验证集 | 10,000 条（BERT 全量评估；LLM 抽样 200 条，`seed=42`） |
| 类别数 | 15 |

数据特点：标题极短（均值约 22 字），类别不均衡明显。评估时除 **Accuracy** 外，应重点关注 **Macro F1**（小类与大类等权）。

## 三、实验设置

### 3.1 BERT 微调（`src/train.py`）

- 基座：`bert-base-chinese`（约 110M 参数，全量微调）
- 默认：3 epoch，`batch_size=32`，`max_length=128`，AdamW 分层学习率（BERT 2e-5 / 分类头更高）
- 验证指标：`val_acc`、`val_macro_f1`（记录在 `outputs/train_log_*.json`）

### 3.2 LLM 方案（`src_llm/`）

- 基座：`Qwen2-0.5B-Instruct`
- **Zero-shot**：Prompt 约束输出 15 类之一，无需训练
- **SFT**：LoRA（r=8），Chat 格式 + 仅在 assistant 回复上计算 loss；训练日志见 `outputs/train_log_sft.json`
- 评估：验证集随机 **200 条**，结果见 `outputs/llm_*_results.json`

## 四、实验结果汇总

### 4.1 总表对比

| 方法 | 评估集 | Accuracy | Macro F1 / 备注 | 无法解析 |
|------|--------|----------|-------------------|----------|
| BERT · CLS 池化（普通 Loss） | val 全量 10k | **56.18%** | **0.5500** | 0 |
| BERT · CLS 池化（加权 Loss） | val 全量 10k | **55.93%** | **0.5534** | 0 |
| BERT · Mean 池化 | val 全量 10k | **56.57%** | **0.5489** | 0 |
| BERT · Max 池化 | val 全量 10k | **55.80%** | **0.5538** | 0 |
| LLM Zero-shot（Qwen2-0.5B） | val 抽样 200 | **36.00%** | — | **58 条（29.0%）** |
| LLM SFT · LoRA（Qwen2-0.5B） | val 抽样 200 | **58.00%** | — | 2 条（1.0%） |

### 4.2 BERT 训练过程

> Mean/Max 本次各训练 **1 epoch**（`batch_size=16`），与 CLS 的 3 epoch 对比时需注意训练轮数差异。

**CLS + 普通 Loss**（`outputs/train_log_cls.json`）：

| Epoch | Train Acc | Val Acc | Val Macro F1 | 耗时(s) |
|-------|-----------|---------|--------------|---------|
| 1 | 51.41% | 55.26% | 0.5183 | 666 |
| 2 | 60.98% | 56.00% | 0.5276 | 696 |
| 3 | 68.90% | **56.18%** | **0.5500** | 654 |


**Mean / Max 池化**（各 1 epoch，`batch_size=16`）：

| 池化 | Val Acc | Val Macro F1 | 耗时(s) |
|------|---------|--------------|---------|
| Mean | **56.57%** | 0.5489 | 1272 |
| Max  | 55.80% | **0.5538** | 1263 |

- **Mean**：Val Acc 在三种池化中最高，Macro F1 略低于 CLS/Max。
- **Max**：Macro F1（0.5538）接近加权 CLS（0.5534），在不均衡场景下对少数类更友好。

**CLS + 加权 Loss**（`outputs/train_log_cls_weighted.json`）：

| Epoch | Train Acc | Val Acc | Val Macro F1 | 耗时(s) |
|-------|-----------|---------|--------------|---------|
| 1 | 49.88% | 54.17% | 0.5336 | 339 |
| 2 | 59.92% | 55.24% | 0.5435 | 339 |
| 3 | 66.87% | **55.93%** | **0.5534** | 339 |

**观察：**

- 普通 Loss 的 **Val Acc 略高**（56.18% vs 55.93%），但 **Macro F1 略低**（0.5500 vs 0.5534）。
- 在不均衡场景下，**Accuracy 会“掩盖”小类表现**；加权 Loss 牺牲少量整体准确率，换取更均衡的类别 F1，与课程设计一致。
- 加权版单 epoch 耗时更短，可能与 GPU 缓存、I/O 或当时负载有关，不影响结论方向。

### 4.3 LLM 结果（200 条抽样，可直接对比）

| 指标 | Zero-shot | SFT (LoRA) | 相对提升 |
|------|-----------|------------|----------|
| Accuracy | 36.0% (72/200) | 58.0% (116/200) | **+22.0 pp** |
| 无法解析 | 58 (29.0%) | 2 (1.0%) | 显著降低 |

**SFT 训练 Loss**（`outputs/train_log_sft.json`）：

| Epoch | Train Loss | Val Loss |
|-------|------------|----------|
| 1 | 0.732 | 0.659 |
| 2 | 0.611 | **0.639**（最优） |
| 3 | 0.518 | 0.652（回升，略过拟合） |

第 3 epoch 验证 loss 上升，说明 **早停（epoch=2）** 可能更优；当前 SFT 评估准确率 58% 已与 BERT CLS（~56%）同量级，且仅训练约 0.22% 参数（LoRA）。

### 4.4 Zero-shot 典型失败模式（定性）

从 `outputs/llm_zero_shot_results.json` 可见：

1. **标签不在词表内**：模型输出「外交」「政治」「房地产」「时尚」等，无法映射到 15 类 → `pred_label=null`，占失败大头。
2. **近义类别混淆**：如「电竞」→「体育/娱乐/游戏」，「国际」→「军事/外交」。
3. **标题过短、歧义大**：单看标题难以区分财经/科技/房产等。

SFT 通过 **固定输出格式 + 在标注数据上对齐标签空间**，将「无法解析」从 29% 压到 1%，是 Zero-shot 到可用分类器的关键一步。

## 五、方法对比与结论

### 5.1 准确率与成本

```
准确率（val）
  LLM SFT (58%, 200条) > BERT Mean (56.57%, 10k) ≈ BERT CLS (56.18%, 10k) > BERT Max (55.80%, 10k) >> LLM Zero-shot (36%, 200条)

Macro F1（BERT, 10k）
  CLS+加权 (0.5534) ≈ Max (0.5538) > CLS (0.5500) > Mean (0.5489)
```

| 维度 | BERT fine-tune | LLM Zero-shot | LLM SFT |
|------|----------------|---------------|---------|
| 训练成本 | 中（全量 110M，约 30+ min/epoch GPU） | **无** | 低（LoRA 1.08M 可训练参数） |
| 数据需求 | 需全量标注 | 仅需 Prompt | 需标注（项目用 5k 子集亦有效） |
| 推理稳定性 | 稳定（softmax 15 类） | 差（生成式 + 解析） | 较好（仍依赖生成） |
| 小类/不均衡 | 可用加权 Loss 调 Macro F1 | 易忽略长尾类 | 依赖训练分布 |
| 适用场景 | 工业判别式分类 baseline | 快速原型、无标注试探 | 生成式底座 + 少量标注 |

### 5.2 核心结论

1. **BERT 微调仍是稳健 baseline**：在 TNEWS 全量 val 上达到约 **56% Acc / 0.55 Macro F1**，输出稳定、无解析失败。
2. **三种池化（cls/mean/max）**：Mean **Acc 最高**（56.57%）；Max **Macro F1 最高**（0.5538，与加权 CLS 相当）；默认 CLS 综合均衡。池化选择对最终指标影响约 **0.5~1 个百分点**，小于训练轮数与 Loss 加权策略的影响。
3. **类别加权 Loss**：Val Acc 略降、**Macro F1 略升**，说明在不均衡数据上应同时汇报 Acc 与 Macro F1，必要时用 `compare_class_weight.py` 看 per-class Recall。
4. **LLM Zero-shot 不适合直接上线**：0.5B 模型 + 开放生成导致 **29% 无法解析**，准确率仅 36%。
5. **LLM SFT 性价比突出**：少量 LoRA 微调即可将准确率拉到 **58%**，接近 BERT，并极大改善标签合规性；适合「已有生成模型、标注有限」的场景。
6. **模型规模上限**：Qwen2-0.5B 与 bert-base-chinese 参数量级不同，若要超越 BERT 明显幅度，需更大模型或 API 级 LLM，本实验主要对比 **范式差异** 而非 SOTA 绝对值。

## 六、建议与后续工作

1. **（可选）统一 epoch**：Mean/Max 当前为 1 epoch，若需与 CLS 公平对比可再跑 `--epochs 3`。
2. **BERT 评估细化**：`evaluate.py --pool cls`，查看「证券」等少类的 Recall；运行 `compare_class_weight.py` 生成 `compare_class_weight.png`。
3. **LLM 早停**：SFT 以 val_loss 最低 epoch（当前为 2）做 checkpoint，避免第 3 epoch 过拟合。
4. **统一评估口径**：LLM 200 条与 BERT 10k 条口径不同；若需严格对比，可对同一 200 条子集跑 `evaluate.py` / `evaluate_sft.py`。
