# 第十一周作业：天气查询工具调用改造为循环调用

## 一、作业题目

把 `function_call_mcp_cli` 项目中的**天气查询工具调用**，从**单轮闭环**改造为**循环调用**（ReAct Agent）。

## 二、核心改造点

### 单轮闭环（原版）

```
用户提问 → LLM（可能输出 tool_calls）→ 执行工具 → LLM 生成最终回答 → 结束
```

模型**最多只能调一轮工具**。若问题需要「先查 A 城市，再根据结果查 B 城市」，单轮模式无法继续调工具。

### 循环调用（本作业）

```
用户提问 → while 循环:
              LLM 请求
              若仍有 tool_calls → 执行 → 回填 messages → 继续
              若无 tool_calls → 跳出，content 即最终回答
```

模型可以**多轮**调用 `get_weather`，适合多城市对比、链式条件查询等场景。

### 代码对比（关键差异）

**单轮**（`run_single_round.py`）：

```python
resp = client.chat.completions.create(..., tools=TOOLS_SCHEMA)
if msg.tool_calls:
    # 执行工具一次
    resp = client.chat.completions.create(...)  # 直接结束
answer = msg.content
```

**循环**（`run_weather_loop.py`）：

```python
while rounds_used < max_rounds:
    resp = client.chat.completions.create(..., tools=TOOLS_SCHEMA)
    if not msg.tool_calls:
        break  # 模型认为信息够了
    # 执行工具并回填，继续下一轮
```

## 六、预期效果

### 示例 1：单城市（单轮/循环结果相近）

```
Q：宁德今天天气怎么样？
→ 1 次 get_weather → 输出宁德当前温度与 3 日预报
```

### 示例 2：多城市对比（循环优势明显）

```
Q：分别查询北京、上海、广州的天气，告诉我今天哪个最热？

循环模式可能输出：
  ── 第 1 轮 ── → get_weather(北京)
  ── 第 2 轮 ── → get_weather(上海)
  ── 第 3 轮 ── → get_weather(广州)
  ── 第 4 轮 ── → 无 tool_calls，汇总对比

单轮模式：模型一次并行调 3 个城市也能完成，但若模型选择逐个查询则单轮不够。
```

### 示例 3：链式条件（必须循环）

```
Q：先查深圳天气，如果今天有雨再查东莞天气对比

循环模式：
  第 1 轮查深圳 → 第 2 轮根据结果决定是否查东莞 → 第 3 轮总结
```

## 八、技术要点

1. **循环终止条件**：`msg.tool_calls` 为空时跳出；或达到 `max_rounds` 上限强制收尾
2. **消息回填**：每轮 assistant 的 tool_calls 消息 + 各 tool 结果必须按 OpenAI 格式完整保留
3. **工具复用**：直接 import 教学项目的 `get_weather`，保证 geocoding 歧义处理一致
4. **安全上限**：`--max-rounds` 防止模型无限循环调工具
