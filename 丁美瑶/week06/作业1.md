# 文本分类不同训练方法效果对比分析报告

---

## 一、实验概述

本报告基于 TNEWS（CLUE 基准）数据集，对三种文本分类训练方法进行对比分析：
- **BERT Fine-tuning**：传统判别式微调方法
- **LLM Zero-shot**：大模型零样本分类
- **LLM SFT + LoRA**：指令微调结合高效参数微调

---

## 二、方法原理对比

| 维度 | BERT Fine-tuning | LLM Zero-shot | LLM SFT + LoRA |
|------|-----------------|---------------|----------------|
| **核心思想** | 冻结预训练模型，微调分类头 | 利用大模型通用知识，通过 prompt 引导分类 | 在通用大模型基础上进行领域适配 |
| **训练方式** | 全量参数训练（110M） | 无需训练 | 仅训练 LoRA 参数（1.08M） |
| **数据需求** | 需要标注数据 | 无需标注数据 | 需要标注数据（可少量） |
| **Prompt 设计** | 无需 | **关键因素** | 需要构建 chat 格式数据 |

---

## 三、实验结果对比

### 3.1 核心指标对比

| 训练方法 | 验证集准确率 | Macro F1 | 未解析样本数 |
|---------|------------|----------|-------------|
| BERT Fine-tuning (cls) | **56.18%** | 55.0% | - |
| BERT Fine-tuning (加权loss) | 55.93% | **55.3%** | - |
| LLM SFT (LoRA) | **58.0%** | - | 2 |
| LLM Zero-shot | 36.0% | - | **58** |

### 3.2 训练成本对比

| 训练方法 | 可训练参数 | 训练时间（3 epoch） | 硬件需求 |
|---------|----------|-------------------|---------|
| BERT Fine-tuning | 110M (100%) | 30~90 min (CPU) | CPU/GPU均可 |
| LLM Zero-shot | 0 | 无需训练 | GPU |
| LLM SFT + LoRA | 1.08M (0.22%) | ~18 min (RTX 4060) | 需要GPU |

### 3.3 训练过程分析

**BERT Fine-tuning（cls池化）训练曲线：**
- Epoch 1: 训练准确率 51.4% → 验证准确率 55.3%
- Epoch 2: 训练准确率 61.0% → 验证准确率 56.0%
- Epoch 3: 训练准确率 68.9% → 验证准确率 56.2%

**LLM SFT（LoRA）训练曲线：**
- Epoch 1: 训练损失 0.73 → 验证损失 0.66
- Epoch 2: 训练损失 0.61 → 验证损失 0.64（最优）
- Epoch 3: 训练损失 0.52 → 验证损失 0.65（轻微过拟合）

## 四、方法优缺点分析

### 4.1 BERT Fine-tuning

**优点：**
- 成熟稳定，工业界广泛应用
- 训练速度快，CPU即可运行
- 分类头结构清晰，易于调试

**缺点：**
- 性能上限受限于模型规模
- 需要较多标注数据
- 泛化能力有限

### 4.2 LLM Zero-shot

**优点：**
- 完全无需训练，即时推理
- 无需标注数据，快速上手
- 利用大模型的通用知识

**缺点：**
- 准确率最低（36%），效果不稳定
- 输出格式不可控（58条未解析）
- 严重依赖 prompt 设计质量

### 4.3 LLM SFT + LoRA

**优点：**
- 准确率最高（58%），效果最优
- 仅训练 0.22% 参数，高效经济
- 保留大模型能力的同时适配领域数据

**缺点：**
- 需要 GPU 资源
- 训练流程相对复杂
- 需要一定标注数据

## 五、典型错误案例分析

### Zero-shot 失败案例：
```
文本："小产权3500和大产权6500的房子，应该选哪个？"
真实标签：农业 | 模型输出：房地产（未匹配到标签）
原因：标签体系差异，模型输出"房地产"而非数据集定义的"房产"
```

### SFT 失败案例：
```
文本："步枪之王M416枪械搭配及实战运用"
真实标签：电竞 | 模型输出：军事
原因：M416在游戏和现实中均有指代，歧义导致分类错误
```

## 六、结论与建议

### 6.1 核心结论

1. **效果排序**：LLM SFT (58%) > BERT Fine-tuning (56%) > LLM Zero-shot (36%)
2. **效率排序**：LLM Zero-shot（无需训练）> BERT Fine-tuning (CPU) > LLM SFT (GPU)
3. **数据需求**：Zero-shot < SFT < Fine-tuning

### 6.2 适用场景建议

| 场景 | 推荐方法 |
|------|---------|
| **快速原型验证**、无标注数据 | LLM Zero-shot |
| **资源受限环境**、CPU部署 | BERT Fine-tuning |
| **追求最佳效果**、有GPU资源 | LLM SFT + LoRA |

### 6.3 改进方向

1. **Zero-shot**：优化 prompt 设计，增加标签映射逻辑
2. **Fine-tuning**：尝试 mean/max 池化策略，调整学习率
3. **SFT**：增加训练数据量，调整 LoRA 秩参数

## 附录：标签分布说明

TNEWS 数据集包含15个类别，存在明显类别不均衡：
- **大数据类**：科技（5955条）、财经（4731条）
- **小数据类**：证券（257条）、农业（457条）

加权 loss 策略对小类别 Macro F1 有轻微提升（+0.3%），但整体准确率略降。
