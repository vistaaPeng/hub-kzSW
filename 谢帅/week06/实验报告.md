# 中文文本分类实验报告

> 数据集：CLUE/TNEWS（15 类，训练 53360 条，验证 10000 条）
> 硬件：AutoDL RTX 3090 24GB / PyTorch 2.7.0 / CUDA 12.8

---

## 一、实验总览

本报告对比了 6 种方法在 TNEWS 新闻标题分类任务上的表现：

| # | 方法 | 模型 | 训练数据量 | 训练参数量 | 每轮耗时 |
|---|------|------|-----------|-----------|---------|
| 1 | BERT Fine-tune (CLS) | bert-base-chinese | 53K | 102.3M | ~156s |
| 2 | BERT Fine-tune (Mean) | bert-base-chinese | 53K | 102.3M | ~156s |
| 3 | BERT Fine-tune (Max) | bert-base-chinese | 53K | 102.3M | ~156s |
| 4 | BERT Fine-tune + 加权Loss | bert-base-chinese | 53K | 102.3M | ~156s |
| 5 | LLM Zero-Shot | Qwen2.5-0.5B-Instruct | 0 | 0 | — |
| 6 | LLM SFT (LoRA) | Qwen2.5-0.5B-Instruct | 5K | 1.08M (0.22%) | ~300s |
| 7 | LLM SFT (全量微调) | Qwen2.5-0.5B-Instruct | 5K | 495M (100%) | ~1380s |

## 二、核心结果对比

### 2.1 验证集准确率与 Macro F1

| 方法 | Val Accuracy | Val Macro F1 | 无法解析率 |
|------|-------------|-------------|-----------|
| BERT CLS | **0.5669** | 0.5599 | 0% |
| BERT Mean | 0.5646 | **0.5643** | 0% |
| BERT Max | 0.5699 | 0.5547 | 0% |
| BERT CLS + Weighted | 0.5603 | 0.5543 | 0% |
| LLM Zero-Shot | 0.3000 | — | 43.0% |
| LLM LoRA SFT | 0.5520 | — | 0% |
| LLM 全量微调 | (val_loss=0.565) | — | 0% |

> 注：LLM SFT 评估使用 500 条随机采样（seed=42），BERT 评估使用全部 10000 条验证集。LLM Zero-Shot 使用 200 条。

### 2.2 关键发现

1. **三种池化策略差异很小**：CLS (56.69%) / Mean (56.46%) / Max (56.99%)，差异仅 ~0.5%。Max 池化略优，但 CLS 是 BERT 论文原始方案，仍是主流选择。

2. **加权 Loss 反而略降整体准确率**：0.5603 vs 0.5669（降 0.66%）。这是因为加权 Loss 提升少数类 Recall 的代价是降低多数类 Precision。Macro F1 也略降（0.5543 vs 0.5599）。详见 [compare_class_weight.png](figures/compare_class_weight.png) 中各类别 Recall 变化。

3. **Zero-Shot 准确率极低 (30%)**，但 43% 无法解析是主因。在能解析的 114 条中，正确率 = 60/114 ≈ 52.6%，说明模型分类能力本身不差，问题在于小模型不遵守输出格式指令。

4. **LoRA SFT 消除格式问题**：无法解析率从 43% → 0%，准确率从 30% → 55.2%。仅用 5K 条数据、0.22% 可训练参数，18 分钟就接近 BERT 53K 全量微调的效果。

## 三、训练过程详细分析

### 3.1 BERT Fine-tune 各池化策略

| Epoch | CLS train_loss → val_acc | Mean train_loss → val_acc | Max train_loss → val_acc |
|-------|-------------------------|-------------------------|-------------------------|
| 1 | 1.436 → 0.5607 | 1.490 → 0.5592 | 1.436 → 0.5631 |
| 2 | 1.045 → 0.5583 | 1.067 → 0.5635 | 1.053 → 0.5616 |
| 3 | 0.817 → **0.5669** | 0.859 → **0.5646** | 0.835 → **0.5699** |

观察：
- Epoch 2 val_acc 出现轻微下降（CLS: 0.5607 → 0.5583），说明存在轻微过拟合
- Epoch 3 恢复并超过 Epoch 1，最终三个池化结果接近

### 3.2 加权 Loss vs 普通 Loss (CLS)

| Epoch | 普通 Loss val_acc | 加权 Loss val_acc |
|-------|-----------------|-----------------|
| 1 | 0.5607 | 0.5453 |
| 2 | 0.5583 | 0.5525 |
| 3 | 0.5669 | 0.5603 |

加权 Loss 整体准确率略低，但对少数类（如"证券"仅 45 条验证样本）的 Recall 有提升。详见 [compare_class_weight.png](figures/compare_class_weight.png)。

### 3.3 LLM LoRA SFT

| Epoch | train_loss | val_loss | 耗时 |
|-------|-----------|---------|------|
| 1 | 0.687 | **0.624** (最优) | ~300s |
| 2 | 0.443 | 0.690 | ~300s |
| 3 | 0.210 | 0.980 | ~300s |

- **明显的过拟合趋势**：train_loss 从 0.687 急降至 0.210，val_loss 从 0.624 升至 0.980
- 最优 checkpoint 在 Epoch 1（val_loss=0.624），已自动保存
- LoRA 可训练参数仅 1.08M (0.22%)，18 分钟完成训练

### 3.4 LLM 全量微调

| Epoch | train_loss | val_loss | 耗时 |
|-------|-----------|---------|------|
| 1 | 0.643 | 0.603 | ~1380s (~23min) |
| 2 | 0.561 | 0.596 | ~1380s |
| 3 | 0.490 | **0.565** (最优) | ~1410s |

- **过拟合程度远低于 LoRA**：train_loss 平稳下降，val_loss 持续改善
- 最优 checkpoint 在 Epoch 3（val_loss=0.565），优于 LoRA 的 0.624
- 耗时约 LoRA 的 4.6 倍（23min vs 5min/epoch），因为更新全部 495M 参数
- 显存需求 ~10GB，RTX 3090 24GB 完全足够

## 四、方法深度对比

### 4.1 判别式 (BERT) vs 生成式 (LLM)

| 维度 | BERT Fine-tune | LLM SFT (LoRA) |
|------|---------------|----------------|
| 分类范式 | 判别式：直接输出 logits | 生成式：输出类别名文本 |
| 无法解析率 | 0%（确定性的 argmax） | 0%（SFT 后） / 43%（Zero-Shot） |
| 训练数据 | 53K 全量 | 5K（默认） |
| 训练时间 | ~8min (3 epochs) | ~18min (3 epochs) |
| 推理速度 | ~3ms/条 (batch) | ~60ms/条 (逐条生成) |
| 扩展性 | 需修改分类头（类别数变化） | 仅改 prompt 即可适配新任务 |

### 4.2 LoRA vs 全量微调

| 维度 | LoRA (r=8) | 全量微调 |
|------|-----------|---------|
| 可训练参数 | 1.08M (0.22%) | 495M (100%) |
| 最优 val_loss | 0.624 (Epoch 1) | 0.565 (Epoch 3) |
| 过拟合风险 | 高（Epoch 2 即恶化） | 低（3 epochs 持续改善） |
| 训练速度 | ~300s/epoch | ~1380s/epoch |
| 显存需求 | ~3GB | ~10GB |
| 学习率 | 2e-4 | 2e-5 |
| checkpoint 大小 | 仅 adapter (~4MB) | 完整模型 (~1GB) |

> LoRA 的核心优势：极少参数即可达到接近 BERT 的分类准确率（55.2%），适合资源受限场景。全量微调 val_loss 更低（0.565 vs 0.624），理论上上限更高，但过拟合风险需要更多数据来缓解。

## 六、总结

| 结论 | 说明 |
|------|------|
| BERT Fine-tune 是当前最优方案 | 56.99% accuracy，推理快，无格式问题 |
| 三种池化差异极小 | 实践中 CLS 即可，不必纠结池化选择 |
| 加权 Loss 对少数类有帮助但整体略降 | 适合少数类 Recall 重要的场景 |
| LLM Zero-Shot 格式遵守率极低 | 0.5B 模型不适合直接做分类任务 |
| LoRA SFT 高效实用 | 仅 5K 数据 + 0.22% 参数 → 55.2% accuracy |
| 全量微调 val_loss 更优 | 但需更多训练数据防止过拟合，性价比不如 LoRA |

> 进一步提升方向：1) 使用更大 LLM（Qwen2.5-7B）做 SFT；2) LoRA 增加训练数据（--num_train -1）；3) BERT 增加训练轮数或调整学习率。
