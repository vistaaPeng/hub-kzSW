**3.2 标签体系构建** (第29-38行)

```python
# 修改前：硬编码 10 种实体类型
ENTITY_TYPES = ["address", "book", "company", ...]
labels = ["O"] + [f"B-{e}" for e in ENTITY_TYPES] + [f"I-{e}" for e in ENTITY_TYPES]

# 修改后：从 label_names.json 读取
def build_label_schema(data_dir: Optional[Path] = None):
    d = data_dir or DATA_DIR
    with open(d / "label_names.json", "r", encoding="utf-8") as f:
        labels = json.load(f)
    label2id = {lbl: i for i, lbl in enumerate(labels)}
    id2label = {i: lbl for lbl, i in label2id.items()}
    return labels, label2id, id2label
```

## 三、修改思路

### 核心差异：两种数据集格式对比

| 维度 | cluener (旧) | peoples_daily (新) |
|-----|-------------|-------------------|
| **标注格式** | span 格式（实体位置） | 逐 token 标签 |
| **输入字段** | `text` + `label` (dict) | `tokens` + `ner_tags` (list) |
| **标签示例** | `{"name": {"张三": [[0,1]]}}` | `["B-PER", "I-PER", "O", ...]` |
| **实体类型数** | 10种 | 3种（PER/ORG/LOC） |

### 修改逻辑

1. **路径切换**：三处 `DATA_DIR` 统一指向新数据集目录

2. **标签体系适配**：
   - cluener: 动态生成 BIO 标签（O + B-xxx + I-xxx）
   - peoples_daily: 直接读取 `label_names.json` 中的标签列表

3. **Dataset 重构**：
   - 移除 `span_to_bio()` 函数（不再需要）
   - 移除 `CluenerDataset` 类
   - 新增 `PeoplesDailyDataset` 类，直接处理 `tokens` 和 `ner_tags` 字段

## 四、标签体系对比

| cluener (10种实体) | peoples_daily (3种实体) |
|-------------------|-----------------------|
| address, book, company | PER (人物) |
| game, government, movie | ORG (组织) |
| name, organization, position | LOC (地点) |
| scene | - |

## 五、运行方式

修改完成后，训练命令保持不变：

```bash
python train.py              # BERT+Linear 基线模型
python train.py --use_crf    # BERT+CRF 模型
python evaluate.py           # 在验证集评估
```
