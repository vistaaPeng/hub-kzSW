# 中文文本分类项目 · 完整解读与对比分析

> 基于 CLUE/TNEWS 15 类新闻标题分类任务，对比 BERT Fine-tune、LLM Zero-shot、LLM SFT + LoRA 三种范式。

---

## 一、项目概览

### 1.1 任务定义

给定一条中文短新闻标题，预测其所属类别（共 **15 类**）：

| ID | 类别 | ID | 类别 |
|----|------|----|------|
| 0 | 故事 | 8 | 科技 |
| 1 | 文化 | 9 | 军事 |
| 2 | 娱乐 | 10 | 旅游 |
| 3 | 体育 | 11 | 国际 |
| 4 | 财经 | 12 | 证券 |
| 5 | 房产 | 13 | 农业 |
| 6 | 汽车 | 14 | 电竞 |
| 7 | 教育 | | |

### 1.2 数据集

| 划分 | 条数 | 有无标签 |
|------|------|----------|
| train.json | 53,360 | ✅ |
| val.json | 10,000 | ✅（主评估集） |
| test.json | 10,000 | ❌（CLUE 惯例） |

### 1.3 数据关键特点

| 特点 | 数值 | 对建模的影响 |
|------|------|-------------|
| 文本极短 | 均值 22 字，P99 ≈ 39 字 | `max_length=64` 即可覆盖 100% |
| 中文分词 | Token/字符比 ≈ 0.97 | 中文 BERT 基本 1 字 = 1 token |
| 类别不均衡 | 科技 5955 条 vs 证券 257 条（**23:1**） | 需关注 Macro F1、加权 Loss |

## 三、三种方法详解

### 3.1 方式一：BERT 判别式 Fine-tune

#### 模型结构

```
输入句子
    ↓
BertTokenizer → input_ids, attention_mask, token_type_ids
    ↓
BertModel → last_hidden_state [B, L, 768]
    ↓
池化（cls / mean / max）→ 句向量 [B, 768]
    ↓
Dropout(0.1) → Linear(768 → 15) → logits [B, 15]
    ↓
CrossEntropyLoss（可选类别权重）
```

#### 三种池化策略

| 策略 | 实现 | 直觉 |
|------|------|------|
| `cls` | `last_hidden[:, 0, :]` | 取 [CLS] 位置，BERT 原论文分类方式 |
| `mean` | 有效 token 均值（排除 padding） | 所有词信息的平均表达 |
| `max` | 有效 token 逐维取最大值 | 保留每个维度最显著的激活 |

#### 训练技巧

| 技术 | 做法 | 原因 |
|------|------|------|
| 分层学习率 | BERT: `2e-5`，分类头: `1e-4` | 预训练层小步微调防遗忘；新头需更快收敛 |
| AdamW + weight decay | `weight_decay=0.01` | Transformer 微调事实标准 |
| Linear Warmup + Decay | 前 10% 步 warmup，再线性衰减 | 训练初期稳定，保护预训练知识 |
| 梯度裁剪 | `max_norm=1.0` | 防止梯度爆炸 |
| 梯度累积 | `--grad_accum` | 显存不足时等效增大 batch |
| 加权 Loss | `compute_class_weight("balanced")` | 缓解长尾类被忽略 |
| Checkpoint | 仅保存验证集 accuracy 最高的模型 | 避免过拟合 checkpoint |

#### 使用命令

```bash
cd src

# 默认参数（CLS 池化，3 epoch）
python train.py

# 均值池化 + 加权 loss
python train.py --pool mean --use_class_weight

# 自定义参数
python train.py --pool max --epochs 5 --batch_size 16 --lr 2e-5

# 评估
python evaluate.py --pool cls

# 推理
python predict.py --text "苹果发布了最新的 iPhone 17 系列手机"
```

### 3.2 方式二：LLM Zero-shot 分类

#### 流程

```
val.json 随机采样 N 条
       ↓
构造 messages：[system 说明任务+类别列表, user 标题+"类别："]
       ↓
apply_chat_template → input_ids
       ↓
model.generate(max_new_tokens=8, do_sample=False)  # 贪心解码
       ↓
decode 新生成 token → parse_prediction（子串匹配类别名）
       ↓
与真实标签比较 → 保存结果
```

#### Prompt 设计

```python
SYSTEM_PROMPT = (
    "你是一个新闻标题分类助手。请将给定的新闻标题分类到以下类别之一，"
    "只输出类别名称，不要输出任何其他内容。\n"
    "可选类别：" + "、".join(LABEL_NAMES)
)
# User: "新闻标题：{text}\n类别："
```

#### 关键特点

- **无需训练**：完全依赖预训练 + 指令对齐后的生成能力
- **输出不可控**：模型可能输出"房地产"而非"房产"、"政治"而非"国际"
- **解析容错**：用子串包含做模糊匹配（`if name in raw_output`）

#### 使用命令

```bash
cd src_llm

# 快速演示（5 条）
python classify_llm.py --demo

# 正式评估（200 条）
python classify_llm.py --num_samples 200

# 指定模型路径
python classify_llm.py --model_path "../pretrain_models/Qwen2-0.5B-Instruct"
```

### 3.3 方式三：LLM SFT + LoRA 指令微调

#### 核心概念一：任务改写（分类 → 对话）

```
system:  "你是新闻分类助手，只输出类别名，可选：故事/文化/..."
user:    "新闻标题：{sentence}\n类别："
assistant: "教育"          ← 只有这部分参与 loss 计算
```

#### 核心概念二：Loss Masking（SFT 的灵魂）

```
token:  [system...][user...][assistant\n][科][技][EOS]
label:  [ -100  ...][ -100 ...][ -100  ][id][id][id]
         ↑ 不计算 loss              ↑ 只在这里算 cross-entropy
```

- `-100` 被 `CrossEntropyLoss(ignore_index=-100)` 自动忽略
- **Pretrain**：对所有 token 算 loss
- **SFT**：只对模型应生成的 assistant 部分算 loss

#### 核心概念三：LoRA（Low-Rank Adaptation）

```
原始矩阵 W ∈ R^{d×d}（冻结）
         ↓
输出 += B·A·x，其中 A ∈ R^{r×d}，B ∈ R^{d×r}，r=8 << d=896

Qwen2-0.5B 全参数：495,114,112
LoRA r=8 可训练参数：1,081,344（0.22%）
```

| 配置 | 全参数微调 | LoRA r=8 |
|------|-----------|----------|
| 可训练参数 | 495M（100%） | ~1.08M（0.22%） |
| 推荐学习率 | `2e-5` | `2e-4` |
| 显存需求 | 8~10GB+ | ~3GB |
| Checkpoint | 完整模型 | 仅 adapter + config |

#### 使用命令

```bash
cd src_llm

# LoRA 微调（默认 5000 条，3 epoch）
python train_sft.py

# 全量数据
python train_sft.py --num_train -1

# 调大 LoRA rank
python train_sft.py --lora_r 16

# 全量微调（需显存 ≥ 16GB）
python train_sft.py --full_ft --lr 2e-5

# 评估（三方对比）
python evaluate_sft.py --num_samples 200
```

## 四、三种方法对比

### 4.1 核心对比表

| 维度 | BERT Fine-tune | LLM Zero-shot | LLM SFT (LoRA) |
|------|---------------|---------------|----------------|
| **模型** | bert-base-chinese (110M) | Qwen2-0.5B (500M) | Qwen2-0.5B + LoRA |
| **可训练参数** | 110M (100%) | 0 | 1.08M (0.22%) |
| **训练数据** | 全量 53K 条 | 无需 | 5K 条（仅 9.4%） |
| **训练时间** | ~30min (CPU, 3epoch) | 无 | ~18min (GPU, 3epoch) |
| **推理速度** | ~5ms/条 (GPU) | ~2s/条 (CPU) | ~0.06s/条 (GPU) |
| **实测准确率** | **56.18%** (val) | **36%** (200条) | **58%** (200条) |
| **Macro F1** | **55.34%** | — | — |
| **无法解析** | 0% | **29%** (58/200) | **1%** (2/200) |
| **输出形式** | logits → argmax | 文本生成 → 解析 | 同 Zero-shot |
| **教学重点** | tokenize / pooling / optimizer / loss | prompt 设计 / generate | chat 格式 / loss mask / LoRA |

### 4.2 代码层面差异

| 环节 | BERT | Zero-shot LLM | SFT LLM |
|------|------|---------------|---------|
| 输入构造 | `BertTokenizer` 固定长度 pad | `apply_chat_template` | 同左，且拆 prompt/response |
| 模型前向 | 一次 encode → logits | 无训练；推理 `generate` | 训练时 `CausalLM` + mask loss |
| 得到类别 | `argmax(logits)` | 生成字符串 → 解析 | 同 Zero-shot |
| 损失函数 | `CrossEntropyLoss(logits, label_id)` | 无 | `CrossEntropyLoss` 仅在 response token |
| 失败模式 | 几乎总能给出 15 类之一 | ~29% 无法解析（小模型） | ~1% 无法解析 |

### 4.3 实测训练日志

| 2 | 1.0609 | 0.6098 | 0.5600 | 0.5276 | 696s |
| 3 | 0.8415 | 0.6890 | **0.5618** | **0.5500** | 654s |

## 五、不同参数对结果的影响

### 5.1 池化策略（`--pool cls/mean/max`）

| 策略 | 特点 | 预期差异 |
|------|------|---------|
| `cls` | BERT 原生方式，[CLS] 被预训练为句子摘要 | 默认基线，表现稳定 |
| `mean` | 所有 token 语义平均 | 对语义相似度任务常更优 |
| `max` | 保留最强特征激活 | 对关键词驱动分类可能有微弱优势 |

本数据集文本极短（均值 22 字），三种池化差异不大。建议分别跑一遍对比：

```bash
python train.py --pool cls --epochs 3
python train.py --pool mean --epochs 3
python train.py --pool max --epochs 3
```

### 5.2 类别加权（`--use_class_weight`）

| 指标 | 普通 Loss | 加权 Loss | 变化 |
|------|----------|----------|------|
| val_acc | **0.5618** | 0.5593 | -0.0025 |
| val_macro_f1 | 0.5500 | **0.5534** | +0.0034 |

**解读**：Accuracy 略降但 Macro F1 提升 → 加权 loss 牺牲了一点多数类准确率，但**提升了少数类（如证券）的 Recall**。对于不均衡数据集，Macro F1 是更公平的指标。

### 5.3 训练轮数（`--epochs`）

```
Epoch 1 → Epoch 2 → Epoch 3
train_loss:  1.44 → 1.06 → 0.84   (持续下降)
val_acc:     0.553 → 0.560 → 0.562  (缓慢提升，边际递减)
```

- 1 epoch 即可达到 ~55% 准确率（快速验证流程够用）
- 3 epoch 后 val_acc 仍在提升但幅度减小
- 更多 epoch 可能过拟合，建议 **3~5 epoch**

### 5.4 学习率（`--lr`）

| 设置 | 影响 |
|------|------|
| `lr=1e-5` | 更保守，训练更慢但更稳定 |
| `lr=2e-5`（默认） | BERT 微调标准值，推荐 |
| `lr=3e-5` | 激进，可能破坏预训练权重 |
| `head_lr_mult=5.0`（默认） | 分类头 lr = BERT lr × 5，新层需更快收敛 |

### 5.5 截断长度（`--max_length`）

| max_length | 覆盖率 | 显存影响 |
|-----------|--------|---------|
| 32 | ~95% | 最省 |
| **64**（默认） | **~100%** | 推荐 |
| 128 | 100% | 浪费（文本平均仅 22 字） |

### 5.6 Batch Size 与梯度累积

| 配置 | 等效 batch | 显存 |
|------|-----------|------|
| `--batch_size 32` | 32 | 较大 |
| `--batch_size 8 --grad_accum 4` | 32 | **1/4 显存** |

梯度累积原理：每 `grad_accum` 步才执行一次参数更新，多步梯度累加等效于大 batch。

### 5.7 SFT 专属参数

| 参数 | 默认值 | 影响 |
|------|--------|------|
| `--lora_r` | 8 | rank 越大参数越多，效果略好，显存略增 |
| `--lora_alpha` | 16 | 缩放因子，有效学习率 ≈ lr × alpha/r |
| `--num_train` | 5000 | 训练数据量，-1 用全量 53K |
| `--lr` | LoRA=2e-4, 全量=2e-5 | 全量微调必须更小学习率 |
| `--full_ft` | False | 全量微调需 ≥16GB 显存 |

### 5.8 Dropout（`--dropout`）

| 值 | 影响 |
|---|------|
| 0.0 | 无正则化，可能过拟合 |
| **0.1**（默认） | 轻度正则化，推荐 |
| 0.2~0.3 | 强正则化，小数据集可尝试 |

## 六、方法选型建议

```
┌─────────────────────────────────────────────────────────────────┐
│  BERT Fine-tune:  "我从标注数据中学习分类边界"                   │
│  → 判别式模型，一次前向出 logits，稳定可控                      │
│  → 适用：大量标注数据 + 需要部署快 + 要可解释 logits            │
│                                                                  │
│  LLM Zero-shot:   "我靠预训练知识直接猜"                         │
│  → 零成本，但小模型能力有限，输出不可控                          │
│  → 适用：快速原型 / 零标注 / 探索可行性                         │
│                                                                  │
│  LLM SFT + LoRA:  "我用少量标注数据微调大模型的分类能力"         │
│  → 参数高效，数据效率高，但推理较慢                              │
│  → 适用：标注有限 + 想用生成式模型 + 显存中等                   │
└─────────────────────────────────────────────────────────────────┘
```
